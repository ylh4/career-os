# Stories (STAR)

Behavioral-interview narratives in **STAR** form: Situation, Task, Action, Result.
One story per file. Drawn on by `/interview-prep`. Same provenance discipline applies —
results must be traceable to real outcomes (often the same source as an accomplishment).

Filename: kebab-case by theme — e.g. `conflict-resolution.md`, `ambiguous-project.md`.

## Format

```markdown
# <Story title>

- **Themes:** <comma-separated: leadership, conflict, ambiguity, failure, influence, ...>
- **Situation:** <the setting and stakes>
- **Task:** <what you were responsible for>
- **Action:** <the specific steps you took — first person, concrete>
- **Result:** <the outcome, quantified where possible>
- **Source:** <provenance — links back to an accomplishment or attestation>
- **Reflection:** <what you'd say you learned; optional>
```

## Rules
- Map each story to the **themes** interviewers probe so prep can select the right one.
- The **Result** should connect to a corpus accomplishment wherever it asserts a metric.

Worked examples live in `sample-data/stories/`.
